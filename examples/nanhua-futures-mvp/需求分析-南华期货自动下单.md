# 需求分析 — 南华期货自动下单（Python MVP）

## 一、背景与目标

在南华期货场景下，开发一套「期货自动下单」软件（Python 实现）。首版为 MVP：
不接实盘，仅提供模拟券商适配器（SimBroker）+ 券商适配抽象（BrokerAdapter）；
行情指数（上证指数、深证成指）允许使用 mock / CSV 数据源。

## 二、指标参数（技术分析）

1. K 线均线（MA）：周期 `5 + 6 + 7 + 8 + 9 + 11 + 20`
2. BOLL 布林带：`20 + 2`（周期 20，标准差倍数 2）
3. VOL 成交量：`3 + 100`（短均 3，长均 100）
4. MACD：`14 + 53 + 60`（快线 14，慢线 53，DEA 60）
5. CCI：`89`

## 三、信号规则

### 3.1 空（做空 / 看跌）买入点 —— 以 3 分钟 K 线为主

1. 前五条均线并拢，并拢数据小于 2 个点后，向下穿插 20 日线；
2. 布林带横盘振荡时不做（震荡区过滤）；
3. VOL：白线（短均）上升 + 绿柱倍量；
4. MACD：黄白线靠近零轴，绿柱加长；
5. CCI：在半小时内二次下穿 0 轴；
6. 1 分钟 K 线也同方向下穿 20 日线；
7. 共振过滤：上证指数与深证成指同时位于各自 20 日线下方。

### 3.2 多（做多 / 看涨）买入点 —— 以 3 分钟 K 线为主

1. 前五条均线并拢，并拢数据小于 2 个点后，向上穿插 20 日线；
2. 布林带横盘振荡时不做；
3. VOL：白线上升 + 红柱倍量；
4. MACD：黄白线在零轴之上靠近 0 轴，红柱加长；
5. CCI：在半小时内二次上穿 0 轴；
6. 1 分钟 K 线也上穿 20 日线，且 1 分钟与 3 分钟 K 线上都带有白点；
7. 共振过滤：上证指数与深证成指同时位于各自 20 日线上方。

## 四、风控 —— 买入后止损 15 个点

通常买入后在「第二天」平仓。止损固定为 15 个点，按下列四种情形区分对冲方向：

1. **昨日对冲单**，开单的平仓单是**多单**：止损时平空单，并在买入点位**往上 15 个点**设止损点，开空单进行对冲止损。
2. **昨日对冲单**，平仓单是**空单**：止损单平多单，并在买入点位**往下 15 个点**挂多单平多单，作为反向平仓保护。
3. **当天开单**，开单是**多单**：止损点在开单点位**往下 15 个点**，到达即开空单对冲，放到明天。
4. **当天开单**，开单是**空单**：止损点在开单点位**往上 15 个点**，到达即开多单对冲，放到明天。

## 五、交付物（目录结构）

- `config.yaml` —— 集中配置（指标参数、风控点数、数据源路径、券商参数等）
- `indicators/` —— 指标计算（MA / BOLL / VOL / MACD / CCI）
- `signals/` —— 信号判定（多 / 空，含共振与横盘过滤）
- `risk/` —— 风控与止损对冲（15 点规则、四情形对冲）
- `broker/` —— 券商适配：`BrokerAdapter` 抽象 + `SimBroker` 模拟实现
- `main/` 或 `cli` —— 程序入口，串联数据→指标→信号→风控→下单
- `tests/` —— pytest 单元测试，覆盖指标、信号、风控、券商
- `DELIVERY.md` —— 交付说明（运行方式、限制、后续接实盘的扩展点）

## 六、非功能约束

1. Python 3.10+；单进程、无 Web 界面。
2. 首版不接实盘，仅 SimBroker；指数行情用 mock / CSV。
3. 不依赖 TA-Lib（无 C 库），指标用 pandas / numpy 自行实现。
4. 公共接口尽量用返回值表达结果与错误，便于 headless 测试。
5. 先做架构决策，再按 `indicators/ → signals/ → risk/ → broker/` 垂直切片实现与验证，最后 `main` 集成。
